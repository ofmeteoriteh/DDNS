# DDNS

[![License](https://img.shields.io/badge/license-Apache--2.0-lightgrey?style=flat-square)](LICENSE)
[![Go](https://img.shields.io/badge/Go--9ca3af?style=flat-square)](https://go.dev/)
[![Status](https://img.shields.io/badge/status-stable-blue?style=flat-square)]()

> A minimal, self-hosted DDNS client — Cloudflare only, IPv4 and IPv6.

---

## What Is This?

DDNS is a lightweight command-line tool that automatically updates Cloudflare DNS A / AAAA records with the machine's current public IP address.
It is deliberately minimal: one provider, two record types, standard library only.

---

## Highlights

- **Minimal dependencies** — Cloudflare API interaction uses only the Go standard library; interactive setup is powered by [survey](https://github.com/AlecAivazis/survey)
- **Dual-stack support** — fetches both IPv4 and IPv6 public IPs from multiple single-stack sources, and updates A and AAAA records independently
- **Skip on no-change** — compares current IP against existing DNS record content; skips the API call if nothing changed
- **Cross-platform** — pure Go with no CGO; compiles to Linux / macOS / Windows on amd64 / arm64 / arm / 386

---

## How It Works

```text
getip                cloudflare
  │                      │
  ├─ GetIPv4IP() ──────► DDNS("A", ipv4)
  │                        ├─ GET  /dns_records?name=...&type=A
  │                        ├─ IP changed?
  │                        │    ├─ no  → skip
  │                        │    ├─ yes → PUT /dns_records/{id}
  │                        │    └─ empty → POST /dns_records
  │
  └─ GetIPv6IP() ──────► DDNS("AAAA", ipv6)
                           └─ (same flow)
```

## Quick Start

### 1. Download

Download the appropriate binary for the platform from [Releases](https://github.com/ofmeteoriteh/DDNS/releases).

### 2. Setup

```bash
ddns setup
```

The interactive wizard prompts for API key, zone, DNS entries, and optionally generates systemd service + timer files.
Config is saved to `config.json`.

### 3. Run

```bash
ddns
```

---

## CLI Flags

```text
ddns [flags]          Run DDNS update
ddns setup            Interactive setup wizard

Flags:
  -v, --version       Print version
  -h, --help          Print help
  -c, --config PATH   Config file path (default: config.json)
  --dry-run           Simulate without calling API
```

## Usage by Platform

### Linux (systemd)

```bash
# Install binary
cp ddns /opt/ddns/ddns
chmod +x /opt/ddns/ddns

# Interactive setup (generates service + timer files)
cd /opt/ddns && ./ddns setup

# Install service + timer
cp ddns.service ddns.timer /etc/systemd/system/
systemctl daemon-reload
systemctl enable --now ddns.timer

# Trigger once manually
systemctl start ddns
```

### macOS (crontab)

```bash
# Install binary
cp ddns /usr/local/bin/ddns
chmod +x /usr/local/bin/ddns

# Interactive setup
cd /usr/local/bin && ddns setup

# Schedule via crontab
crontab -e
# Add: */5 * * * * cd /usr/local/bin && ./ddns
```

### Windows (Task Scheduler)

```powershell
# Place binary
# Put ddns-windows-amd64.exe at C:\ddns\ddns.exe

# Interactive setup
cd C:\ddns
.\ddns.exe setup

# Schedule via Task Scheduler
# Create a basic task, trigger every 5 minutes, action: start C:\ddns\ddns.exe
```

## Config Example

`config.json` is generated by `ddns setup` and has the following structure:

```json
{
  "keys": [
    { "name": "main", "key": "<YOUR_CLOUDFLARE_API_TOKEN>" }
  ],
  "zones": [
    { "name": "main-zone", "domain": "<YOUR_DOMAIN>", "zone_id": "<YOUR_ZONE_ID>" }
  ],
  "entries": [
    {
      "name": "<YOUR_RECORD_NAME>.<YOUR_DOMAIN>",
      "zone_id": "<YOUR_ZONE_ID>",
      "key": "<YOUR_CLOUDFLARE_API_TOKEN>",
      "types": ["A", "AAAA"],
      "proxied": false
    }
  ]
}
```

## Security Considerations

DDNS handles sensitive data and crosses a trust boundary between the local machine and Cloudflare.

- **Credentials** — The Cloudflare API token is stored in `config.json` in plain text. Protect this file with appropriate filesystem permissions.
- **Network endpoints** — The client calls the Cloudflare API (`api.cloudflare.com`) and public IP detection endpoints over HTTPS.
- **Persistent state** — `config.json` holds credentials, zone IDs, and record names; it must be writable and persist between runs.
- **Trust boundary** — The API token must have permission to edit DNS records for the target zone. Use a token scoped to the minimum required zone and records.

## Honest Limitations

- Cloudflare only. No support for other DNS providers.
- IPv4 and IPv6 A/AAAA records only. No other record types, no wildcard management, no TTL automation beyond the provider default.
- No webhook, notification, or multi-provider fallback when IP detection fails.
- Requires a writable `config.json` on disk; not designed for ephemeral or container-only deployments without a volume mount.
